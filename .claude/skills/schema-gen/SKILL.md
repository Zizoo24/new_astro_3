# Schema Markup Generator & Validator

Generate or validate schema for `$ARGUMENTS`.

## Available Schema Types

### 1. FAQPage (Required on all service pages)

Read the page's FAQ section and generate:

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Question text here?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Answer text here."
      }
    }
  ]
}
```

**Rules:**
- 6-15 questions per page
- First sentence of answer: direct answer
- Include the page's primary keyword in at least 2 questions
- For Arabic pages: questions and answers in Arabic

### 2. Service (Auto-generated by ServiceLayout)

Verify these fields are present:

```json
{
  "@type": "Service",
  "name": "Service name",
  "description": "Service description",
  "provider": {
    "@type": "Organization",
    "name": "OnlineTranslation.ae"
  },
  "areaServed": {
    "@type": "Country",
    "name": "United Arab Emirates"
  },
  "serviceType": "Translation"
}
```

### 3. HowTo (For process guides)

```json
{
  "@type": "HowTo",
  "name": "How to [process]",
  "step": [
    {
      "@type": "HowToStep",
      "position": 1,
      "name": "Step title",
      "text": "Step description"
    }
  ]
}
```

### 4. BreadcrumbList

Generated from URL structure:

```json
{
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://onlinetranslation.ae/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Category",
      "item": "https://onlinetranslation.ae/category/"
    }
  ]
}
```

### 5. Article (For blog posts)

```json
{
  "@type": "Article",
  "headline": "Article title",
  "author": {
    "@type": "Organization",
    "name": "OnlineTranslation.ae"
  },
  "datePublished": "2026-XX-XX",
  "dateModified": "2026-XX-XX"
}
```

## Validation Checks

When validating existing schema:

- [ ] No duplicate schema types on same page
- [ ] All required fields present
- [ ] URLs are absolute (https://onlinetranslation.ae/...)
- [ ] LocalBusiness schema uses @id reference (in BaseLayout)
- [ ] FAQ questions match actual FAQ section content
- [ ] HowTo steps match actual process section
- [ ] Breadcrumb matches URL hierarchy
- [ ] Arabic pages have Arabic text in schema (not English)

## Implementation

Schema is implemented via:
- `src/components/Schema.astro` — Type-safe structured data component
- `src/layouts/ServiceLayout.astro` — Auto-generates Service + FAQ + HowTo from props
- `src/layouts/BaseLayout.astro` — LocalBusiness + WebSite + AggregateRating (global)
- `src/components/Breadcrumb.astro` — BreadcrumbList

**For new pages using ServiceLayout:** Pass `faqs` and `processSteps` props. Schema auto-generates.

**For standalone schema:** Use `<Schema>` component:
```astro
<Schema type="faq" data={{ items: [...] }} />
```

## Output

Provide the complete JSON-LD block ready to paste, or a validation report with issues.
